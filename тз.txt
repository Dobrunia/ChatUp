2) Messenger MVP v0.1 — Итоговый спек
0) Базовые принципы

Приложение: мессенджер 1:1 (MVP), но проектируем логику так, чтобы позже можно было добавить группы без ломки.

Идентификация пользователя в логике: только по user_id (а не по @username), чтобы смена @username не ломала связи.

Realtime: Broadcast для мгновенности + fallback (подписка на вставки / синхронизация при возврате) для надёжности.

Offline: без очереди отправки. При отсутствии сети — баннер + отправка недоступна/ошибка.

1) Профиль пользователя
Поля и правила

@username:

уникальный

только английские буквы в нижнем регистре: a-z, ничего больше

можно менять сколько угодно раз, если новое значение свободно

display_name:

не уникальный

может быть пустым

требуется санитизация/экранирование (защита от инъекций/XSS)

avatar:

только текущий (историю не храним)

Правила отображения имени и аватара

Отображаемое имя:

если display_name не пуст → показываем его

иначе показываем @username

иначе “User”

Дефолтный аватар:

буквы/инициалы из display_name

если display_name пуст → инициалы из @username

2) Поиск пользователей и старт диалога
Поиск

Поиск пользователей:

по @username (уникальный)

по display_name (неуникальный)

Карточка результата поиска:

avatar

display_name

@username

Старт 1:1 диалога

Писать можно всем (нет “запросов на чат”).

Диалог 1:1 создаётся только при первом отправленном сообщении (нажатие “Отправить”).

В системе должен существовать ровно один диалог на одну пару пользователей (без дублей).

3) Сообщения: типы, вложения, лимиты
Типы сообщений (MVP)

Текст

Фото

Голосовое сообщение

Фото

источник: только галерея

можно выбрать до 4 фото за раз

выполняем клиентское сжатие перед загрузкой

лимит: фото до ~10MB каждое

Голосовые

запись внутри приложения, отдельная кнопка

лимит длительности: до 2 минут

обязателен preview:

можно прослушать

удалить

перезаписать

лимит размера: до ~10MB

Медиа-хранение

Медиа хранится в облачном storage.

Сообщение хранит ссылку и метаданные (тип/размер/длительность/и т.п.).

4) Статусы сообщений

У каждого исходящего сообщения отображается статус:

“Отправляется”

локальный optimistic статус до подтверждения отправки (и/или пока грузится медиа)

“Отправлено”

когда сообщение успешно сохранено (insert успешен)

“Прочитано”

когда собеседник отметил прочтение (см. last_read_at ниже)

пользователь не может сам “прочитать” свои исходящие

Дополнительно (технически, для UX):

статус ошибки/повтора (“не отправилось”) допускается как внутренняя логика и UI-кнопка “Retry”.

5) Read receipts (прочитано)

Механизм: per-conversation last_read_at на участника.

Правило (гибрид C):

при открытии чата:

если пользователь уже находится внизу (видит последние сообщения) → сразу обновляем last_read_at

иначе обновляем last_read_at только когда пользователь доскроллил до низа/последнего сообщения

Непрочитанные в списке чатов:

считаем как messages.created_at > last_read_at

6) Realtime поведение (сообщения, синхронизация, надёжность)
Доставка новых сообщений

Основной путь: broadcast (мгновенно)

Fallback:

подписка на новые записи (insert) / синхронизация при возврате и reconnect

при reconnect подтягиваем последние N сообщений и сверяемся

7) Typing + Voice recording индикаторы
Typing

typing:start — при первом вводе символа

далее отправляем keep-alive события периодически, пока пользователь печатает

typing:stop — когда 3 секунды нет событий (таймаут = 3 сек)

UI в чате: “печатает…”

Voice recording indicator

события “recording:start/stop”

UI: “записывает голосовое…”

8) Presence (онлайн)

Presence = пользователь online, пока приложение активно (любой экран).

В списке чатов показываем online-индикатор собеседника.

9) Push-уведомления

Push отправляем всегда на любое входящее новое сообщение:

даже если чат открыт

но не отправляем пуш на свои собственные исходящие

Текст пуша:

для текста: Новое сообщение от {name}: {text}

для медиа (если нужно единообразие): Новое сообщение от {name}: Фото / Голосовое сообщение

Звук/вибрация: должны быть (системные достаточно; тумблеры в настройках управляют поведением приложения)
