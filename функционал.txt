MVP (чтобы уже можно было пользоваться)
0) Онбординг и аккаунт

Экран приветствия

Авторизация (один путь для простоты):

по номеру телефона + код (лучше для “найти человека”)

Мини-профиль:

имя - то что отображается у других пользователей

юзернейм (@handle) стартого генерируется автоматически (можно поменять в профиле) уникальный у каждого пользователя нужно для поиска или в дальнейшем чтобы тегать челочека

аватар (опционально)

1) Поиск человека

Поиск по:

@username

телефону (если разрешено)

внутреннему ID

Экран “профиль пользователя”:

имя/аватар

кнопка Написать

кнопка Заблокировать (минимум безопасности)

2) Личные диалоги

Список диалогов:

аватар, имя

последнее сообщение

бейдж непрочитанных

время

Создание диалога автоматически при первом сообщении

3) Чат (основа)

Отправка:

текст

фото

видео

аудио (запись голосового)

файл (док/архив/и т.п., небольшой)

Получение в real-time (WebSocket)

Статусы:

“отправлено” (сервер принял)

“доставлено” (клиент получил)

“прочитано” (открыт чат)

Индикатор “печатает…” (по желанию, но легко)

Просмотр медиа:

фото/видео полноэкранно

скачивание файла

4) Минимальные настройки

Профиль (изменить имя/юзернейм(если не зянят)/аватар)

Настройки:

уведомления on/off (пока только флаг)

тема (светлая/тёмная) — опционально

Выход из аккаунта

5) Уведомления (минимум)

В мобилке (Capacitor): push на новое сообщение

В PWA: можно позже или ограниченно

V1 (после MVP, когда “уже работает”)
Сообщения

Ответ (reply)

Пересылка (forward)

Удалить “у меня” / “у всех” (с таймером или без)

Редактирование текста (опционально)

Реакции (эмодзи) — опционально

Качество медиа

Сжатие фото/видео на клиенте (ограничение по размеру)

Прогресс загрузки/отправки

Повтор отправки при ошибке

Надёжность

Очередь исходящих (если сеть пропала)

Оффлайн-кеш последних диалогов/сообщений (IndexedDB/SQLite)

Безопасность/антиспам

Жалоба на пользователя

Ограничение частоты сообщений новым контактам

“Кто может писать мне”:

все / только по ссылке / только по username

Позже (если захочется “как у больших”)

Группы

Звонки (WebRTC)

E2E шифрование (сложно, но реально)

Стикеры, боты, каналы

Синхронизация устройств (веб/десктоп)

UX-экраны (минимальный набор)

Splash / Welcome

Login (код/пароль)

Create profile

Chats list

Search user

User profile

Chat screen

Settings

Ограничения по файлам (чтобы сразу было понятно)

Для MVP я бы зафиксировал:

фото: до 10–15 МБ

видео: до 50–100 МБ (или меньше)

файл: до 25–50 МБ

голосовое: до 2–5 минут (или до 10 МБ)
